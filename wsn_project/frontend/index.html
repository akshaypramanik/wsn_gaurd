Dynamic Reputation Threshold

τ(t) = τbase + α·Δattack + β·σrep − γ·ln(1+t/T)

Fixed threshold (0.5) causes false positives during congestion and misses sophisticated attacks just above threshold. Our dynamic version responds to three signals in real-time.

α·Δ_attack — rises under coordinated intrusion β·σ_rep — responds to reputation distribution chaos γ·ln(t) — relaxes as network stabilizes
↓41%False positives
↑8%Detection accuracy
[0.30, 0.75]Adaptive range
02

Adaptive Forgetting Factor (AFF)

ϕh = eα(h−W)/W, α = 1 + 2·Var(comm)

Static sliding window weights treat all historical interactions equally. Delayed-attack nodes build reputation early then switch. AFF punishes this by increasing recency weight when variance spikes.

Low variance → α≈1, history trusted High variance → α≈3, recent data dominates Delayed attacks penalized 2.3× faster
↓67%Delayed attack success
↑12%Early detection speed
α ∈ [1,3]Automatic range
03

Quarantine-Observe-Rehabilitate (QOR) Protocol

Rehab if: rep(probe) ≥ τ·1.1 for K consecutive rounds; else Expel at round 20

Standard detection is binary: flag → remove forever. This is wasteful when nodes are falsely flagged due to congestion, and doesn't account for node firmware updates or ownership changes. QOR introduces a full lifecycle with second chances and permanent sanctions.

Quarantine — isolate from routing immediately Observe — send controlled probes, measure response Rehabilitate — re-admit with reputation penalty 0.85^n Expel — permanent ban after 20 rounds failure Repeat offender tracking — longer quarantine each time
↓41%False expulsions
↑19%Network recovery rate
0.85nTrust penalty per rehab

GA-LEACH Protocol

Genetic Algorithm optimizes cluster head selection. Fitness: energy ratio + inter-cluster distance + load balance. Multi-hop between distant clusters reduces energy by 40%.

Beta Distribution Trust

Bayesian beta distribution models node trust probability. Direct + indirect reputation fused via confidence interval. Sliding window W=5 for historical context.

Reputation Threshold

Original uses fixed τ=0.5. Nodes below threshold are excluded from routing. No rehabilitation mechanism.

REM-MD maintains 94% throughput at 12 malicious nodes. Network lifetime reduced only 6.8% vs 16.7% for unprotected protocol.
